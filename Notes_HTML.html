HTML

      HTML makes up the greater portion of what's on the internet. It includes Text Content, as well as Mark Up that formats some of what you see, and References to other documents. Mark Up in HTML is made up of Tags. Tags are like instructions to tell a browser what Elements to use. Elements that Tags produce can be Inline or Block. Block Elements create an invisible box around the content while Inline Elements only work along single lines.

          INLINE TAGS: b (bold), br (break), em (italic),
a (anchor), img (image), span, strong

BLOCK TAGS: p (paragraph), div, form

HTML Structure

HTML STRUCTURE

      The standard structure of an HTML document is much like a tree. The tree branches off in multiple directions from a central sort of trunk. This structure is sometimes called a DOM or a Document Object Model. HTMLs structure is basically boxes inside of boxes inside of boxes, each box being an Element (Tag). Coders usually use indentation within the structure of the code they are writing. This isn't understood by the computer, but allows the code to have more visual organization for the coder.

Adding CSS Style

CASCADING STYLE SHEETS (CSS)

      CSS controls the style of different HTML Elements. For example, this text is within a paragraph tag, and CSS is applying the font type and the text alignment for paragraph tag.

Back to Top

REPETITION

      Most objectives can be reached with slightly different code. One might apply apply a single CSS font to 10 of their elements and others to 2 elements, when it would have been more efficient to apply that font to a broader element (like the body) and then just apply the 2 other fonts to their proper elements. The first way involves needless repetition which wastes time and increases the likelihood of errors. The problem compounds further when things need to be changed or errors are made, as fixing them will also be more time consuming. Simplifying a lot of style into reusable tags is an example of Abstraction - or reducing something to it's essential characteristics.

Variables and Strings

Python Variables

      Variables are a way to name certain values in Python. This is helpful because it improves code readability, but also functionally, as it allows values to change. For example:

number = 1
          print number * 20

This prints 20, because the variable "number" has taken the value of 1.
Back to Top

Equals Symbol (=)

      Equals symbols have a different meaning in Python than in math. In Python they mean that something will take the value of the other, even if that makes no mathematical sense. For example:

number = 10
              number = number - 5
print number

This prints 5, even though "number" is first defined as 10. The second time "number" is given a value it takes 5 away from the original value of "number", and reassigns that value to the variable "number", thus when we print it we get 10-5, or 5. This is also an example of how the value of a variable can change, as mentioned in the previous concept.
Back to Top

Strings

      In Python, strings are enclosed within single or double quotations. Either work, but the closing and ending type must match. Things within a string will not be recognized as numbers.

Input -> Function -> Output

Functions

      Functions take some input and do something to it, producing an output. A function could, for example, add two numbers no matter what those numbers are.

Back to Top

Parts of a Function

      Functions are started by the keyword def followed by the name of the function. This is then followed by the functions parameters in parenthesis immediately following the functions name, separated by commas and ended with a colon. Next is the body of the function, which is what the function is going to do to the parameters. When printing the result of a function you print the name of the function, followed by the values which are to take the place of the parameters in the function

Functions and Repetition

      Functions help to avoid repetition because it only needs to be defined once. A function which adds two numbers can be called upon any time that two numbers need adding within the code without redefining it.

Return Statements

      Return statements tell Python exactly what a function should produce as output. For example, if the body of a function was that a + b = c then the return statement should either be "return c", or simply put all of it in the return statement like "return a + b".

If, Else, While, and Debugging

If

      If statements are statements used in Python to make code do something different depending upon the result of a comparison. Indented within an if statement is a block which contains what the code will do if the if statement evaluates to true.

Back to Top

Else

      Else statements are used in addition to if statements to tell code what to do if the if statement evaluates to false. Else statements are along the same indentation as the same if statement and also have an indented block within them.

Back to Top

While Loops

      While loops are another Python statement which do something multiple times. You set a condition under which it will continue, and it will continue until the input no longer meets that condition.

Back to Top

Debugging Tips

      If code is causing an error, it's best to start by examining the error message. The last line usually gives the best information to help you figure out the problem. It's also helpful to compare your code to working examples of similar code, and to check that whatever code you may have based yours upon works also. It's most difficult to figure out an issue when the code doesn't produce any error, but doesn't do what you want it to. In many cases it's helpful when debugging something like this to print individual parts of your code to see what isn't working exactly.

Object Oriented Programming

Object Oriented Programming

      Object Oriented Programming is programming based on the concept of Objects and Procedures, often also known as Attributes and Methods. Object Oriented Programming is useful because it allows programmers to understand and create a great deal of complex code without knowing everything about how the objects in their code work - just what they actually do. It encourages the reuse of code, and is an example of Abstraction (REVIEWER - Please use the "Abstraction" link to see an explicit connection between OOP and CSS/HTML).

Modules, Classes, Objects and Instances

      Modules in Python are sort of like containers for code to access with the "." operator. The webbrowser, time, os, and turtle modules are examples of Python modules in the Python Standard Library (a set of files containing procedures that are regularly used in Python and come with Python). Classes are much like modules in that they store code to be accessed with the "." operator, however there will only be one module for a whole program, but many potential instances of a class which don't interfere with each other. Instances are subsequent uses of a class. For example as used in the lesson, there might be a class called movie, and 10 instances of this class, all of which are particular movies. These individual instances of a class are Objects.

Back to Top

Class and Instance Variables

      Instance Variables are variables associated with every instance of a class, and have different copies with different values for each instance. Class Variables are variables that have the same values for all instances, such as the example of movie ratings on a movie website.

Method Overriding

      If a method is defined by the same name in a class and subclass, then the return will depend on whether it's called through the class or subclass. This is called method overriding.
